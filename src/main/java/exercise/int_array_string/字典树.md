# Trie（字典树 / 前缀树）

## 是什么

Trie 是一棵多叉树，从根到某个节点的路径表示一个**前缀**（常见是字符串或数字的逐位/逐字符前缀）。

- 根节点不存有效字符，表示空前缀
- 每一层对应前缀的一个字符（或一位数字）
- 从根走到某节点 = 存在一个以该路径为前缀的串

## 基本结构（Java）

```java
class TrieNode {
    TrieNode[] children = new TrieNode[26]; // 字母：26；数字：10
    boolean isEnd; // 可选：标记某个前缀是否是完整单词/数
}

class Trie {
    TrieNode root = new TrieNode();

    void insert(String word) { /* 沿路径建节点 */ }
    boolean search(String word) { /* 是否存在完整串 */ }
    boolean startsWith(String prefix) { /* 是否存在以 prefix 开头的串 */ }
}
```

数字题可把 `char` 换成 `digit = c - '0'`，`children` 长度为 10。参考：`LongestCommonPrefix.java` 中的 `DigitTrie`。

## 核心操作

| 操作 | 含义 | 复杂度（设串长 L） |
|------|------|-------------------|
| insert | 插入一个串，沿途建节点 | O(L) |
| search | 查询完整串是否存在 | O(L) |
| startsWith / 沿树走 | 查询是否存在某前缀，或能匹配多长 | O(L) |

## 适用场景

- 多个串的**公共前缀**、**前缀匹配**、**前缀统计**
- 词典、自动补全、敏感词过滤
- 数字按位前缀（如 LeetCode 3043：两数组整数的最长公共前缀长度）

## 与 HashSet 前缀法的对比

| | Trie | HashSet 存所有前缀 |
|--|------|-------------------|
| 思路 | 共享前缀路径，不重复存相同前缀 | 每个前缀单独一条记录 |
| 时间 | 插入/查询 O(总位数) | 同量级 |
| 空间 | 节点共享，前缀重叠多时更省 | 字符串对象较多 |
| 选型 | 前缀题模板、需要按位扩展时 | 实现快、位数很少时够用 |

## 写 Trie 题时的步骤

1. **明确「边」是什么**：字母、数字位，还是二进制位
2. **明确要查什么**：完整串？最长匹配前缀？带权统计？
3. **先 insert 谁、再 query 谁**：通常把一侧全部插入，另一侧逐个查询
4. **循环不变量**：当前节点 `node` 代表「已匹配的前缀」；每读一位，若 `children[d] == null` 则无法继续

## 本仓库示例

`LongestCommonPrefix.longestCommonPrefixV3`：

- 将 `arr2` 中每个数插入 Trie（路径即该数的所有前缀）
- 遍历 `arr1`，沿 Trie 走，能走几步即为公共前缀长度
- 复杂度约 O((n + m) × d)，d 为数字位数（本题 d ≤ 9）

## 常见坑

- 忘记区分 **search（完整串）** 与 **startsWith（前缀）**
- 数字 Trie 用 `children[10]`，字母用 `children[26]`，下标别混
- 只 insert 不 query，或 insert/query 的不是同一套「字符集」
- 多叉树层数深时注意空间；本题位数小，Trie 与 HashSet 都能过时限
